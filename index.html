N
Driver Login & Auth
جديد
السائق بيدخل بـ email + password عشان يوصل لمساراته وطلباته.
👤 السائق فقط
ليه جديد؟ الـ 42 مطلوب ما ذكروش خالص، بس من غيره السائق مش قادر يدخل على التطبيق.
N
Active Route View
جديد
صفحة بتعرض المسار المعين للسائق: المحطات بالترتيب، الطلبات، والـ ETA لكل محطة.
👤 السائق فقط
ليه جديد؟ الـ Dispatcher بيعمل المسار، بس محتاج السائق يشوفه — ده الـ interface الناقص.
12
Pre-Trip Safety Inspection
تشيك ليست رقمي إجباري قبل ما السيستم يسمح للسائق يبدأ الرحلة: إطارات، فرامل، أنوار، وثائق.
👤 السائق فقط
الـ flow: السائق يفتح التطبيق ← يكمّل الـ checklist ← بعد كده بس يبقى زرار "Start Route" متاح.
13
Digital Signature Capture
لما العميل يستلم الطرد، السائق بياخد توقيعه الرقمي على الشاشة ويتخزن مشفر مع سجل التوصيل.
👤 السائق يجمع، العميل يوقع
الـ flow: وصل ← مسح QR ← العميل يوقع على الشاشة ← يتحفظ مع الـ timestamp والـ GPS.
14
Photo-Evidence Attachment
في حالة "Safe Drop" لما العميل مش موجود، السائق بياخد صورة لمكان التسليم وبتتخزن مع إحداثيات GPS.
👤 السائق فقط
الـ flow: العميل مش موجود ← Safe Drop ← صورة + GPS تلقائي ← اتحفظ في سجل الطلب.
15
Incident Reporting Workflow
لو حصل حادث أو عطل، السائق بيبلغ من التطبيق ويقدر يطلب سحب أو مساعدة طارئة.
👤 السائق يبلغ ← Dispatcher يستقبل ← Mechanic يتعين
الـ flow: السائق يختار نوع الحادث ← يضغط "Request Tow" ← الـ Dispatcher يشوف تنبيه ← Dynamic Rerouting تشتغل تلقائي.
16
Break & Rest Manager
السيستم بيتابع ساعات شغل السائق وبيلزمه ياخد فترات راحة حسب قوانين العمل.
👤 السائق — تلقائي من السيستم
الـ flow: بعد X ساعات شغل ← السيستم يحجب الـ "Complete Delivery" ← السائق لازم يسجل Break.
17
QR-Code Parcel Verification
قبل ما السيستم يسمح للسائق يكمّل التوصيل، لازم يمسح QR خاص بالطرد للتحقق من الهوية.
👤 السائق فقط
الـ flow: وصل للعميل ← فتح الكاميرا ← مسح الـ QR ← لو صح يفتح زرار "Complete"، لو غلط تنبيه.
18
COD Reconciliation
لو الطلب cash-on-delivery، السائق بيسجل المبلغ المحصّل وبيتتبع في سجل حتى يسلمه للمخزن.
👤 السائق يسجل ← Dispatcher يراجع
الـ flow: العميل يدفع كاش ← السائق يدخل المبلغ ← بيتجمع في ledger ← وقت التسليم للمخزن بيتطابق.
19
Offline Progress Queue
لو النت انقطع، السائق يقدر يكمل شغله عادي، والبيانات بتتخزن محلياً وبتتزامن أوتوماتيك لما النت يرجع.
👤 السائق فقط
الـ flow: نت انقطع ← السيستم يعمل offline mode ← السائق يكمل ← النت رجع ← sync تلقائي.
20
Proximity-Based Auto Check-in
لما السائق يبقى على بُعد 500 متر من العميل، السيستم بيبعتله إشعار تلقائي.
تلقائي — السائق الـ trigger ← العميل يستقبل
الـ flow: GPS السائق يتحرك ← السيستم يحسب المسافة ← أقل من 500م ← إشعار للعميل تلقائي.
21
Return-to-Base (RTB) Logic
لو الطلب مش قادر يتسلّم، السائق بيسجل السبب والطرد بيرجع للمخزن.
👤 السائق يسجل ← Dispatcher يتابع
الـ flow: محاولة فاشلة ← السائق يختار السبب ← الطرد يتحوّل لـ "Returned" ← بيدخل المخزن تاني.
22
Driver Performance Score
السائق يشوف تقييمه المجمّع: سرعة التوصيل + كفاءة الوقود + تقييمات العملاء.
👤 السائق يشوف — Fleet Manager يراجع من الـ Dashboard
المعادلة: (سرعة × وزن) + (وقود × وزن) + (تقييم العميل × وزن) = Score من 100.

السيستم بيعمل رابط مؤقت لكل طلب، العميل يدخل عليه من غير ما يعمل حساب ويشوف موقع السيارة على الخريطة.
🔗 رابط عام — العميل بس يشوف بياناته
الـ flow: الـ Dispatcher يبعت الـ link للعميل ← العميل يفتحه ← خريطة فيها موقع السيارة يتحدث كل X ثواني.
N
Order Status Page
جديد
صفحة فيها timeline خطوة بخطوة: "تم الاستلام من المخزن ← في الطريق ← وصل ← تم التسليم".
👤 العميل فقط
ليه جديد؟ الـ 33 بيغطي الخريطة فقط، بس العميل محتاج يشوف تاريخ الطلب كامل.
34
Delivery Preference Manager
العميل يحدد تعليمات خاصة للتوصيل: "اتركه عند الجار"، "اطرق بهدوء"، "لا تترك بدون توقيع".
👤 العميل يحدد — السائق يشوفه على التطبيق
الـ flow: العميل يفتح الـ link ← يختار نوع التسليم ← السائق يشوف التعليمة على الـ Driver App.
35
Multi-Channel Notifications
العميل بيستقبل إشعارات على SMS أو Email أو Web Push في كل خطوة.
تلقائي من السيستم ← العميل يستقبل
الـ flow: حدث في المسار ← السيستم يختار القناة الأنسب حسب الأولوية ← إشعار للعميل.
13
توقيع الاستلام (من جانب العميل)
العميل بيوقع رقمياً على شاشة السائق لتأكيد الاستلام.
👤 العميل يوقع — السائق بيجمع
ملاحظة: الوظيفة رقم 13 موجودة في الـ Driver App، هنا بس الجانب الخاص بالعميل.
36
Post-Delivery Feedback
بعد التسليم، العميل بيقيّم السائق وحالة الطرد بنجوم وتعليق.
👤 العميل يقيّم — بيأثر على Driver Performance Score
الـ flow: تم التسليم ← إشعار "قيّم تجربتك" ← العميل يبعت التقييم ← بيدخل في الـ score بتاع السائق.

N
Login & Authentication
جديد
Fleet Manager والـ Dispatcher بيدخلوا بـ email + password، السيستم بيعرف role كل واحد ويعرضله الصلاحيات المناسبة.
👤 Fleet Manager + Dispatcher
ليه جديد؟ الـ 42 مش بيتكلم عن آلية الـ login أصلاً.
N
Vehicle CRUD Management
جديد
إضافة سيارة جديدة للأسطول، تعديل بياناتها (نوع، حمولة، لوحة)، أو حذفها.
👤 Fleet Manager فقط
ليه جديد؟ الـ 42 بيتكلم عن صيانة سيارات موجودة بس مش كيف تدخلها في السيستم.
N
User Management (CRUD)
جديد
إضافة سائقين وـ Dispatchers وميكانيكيين، تعديل بياناتهم، وإلغاء حساباتهم.
👤 Fleet Manager فقط
ليه جديد؟ الـ RBAC (37) بيتكلم عن الصلاحيات بس مش عن إدارة المستخدمين نفسهم.
01
Route Priority Balancer
ترتيب الطلبات حسب: هل البضاعة قابلة للتلف؟ مستوى أولوية العميل؟ النافذة الزمنية الموعودة؟
👤 Dispatcher — تلقائي أو يدوي
الـ flow: رفع الطلبات ← السيستم يرتبها بالأولوية ← Dispatcher يراجع ويعدل لو احتاج.
02
Geospatial Clustering
تجميع نقاط التوصيل المتقاربة جغرافياً في "Zones" وتعيين كل Zone لأقرب سيارة أو سائق.
👤 Dispatcher
الـ flow: خريطة بالطلبات ← خوارزمية تجمّعهم ← كل cluster يبقى مسار لسيارة واحدة.
03
Load Capacity Checker
قبل تأكيد المسار، السيستم بيتحقق إن مجموع وزن وحجم الطلبات مش بيعدي الطاقة القصوى للسيارة.
👤 Dispatcher — تلقائي
الـ flow: تعيين طلبات للسيارة ← حساب تلقائي ← لو زاد الحد: تحذير وإيقاف.
04
Dynamic Rerouting
لو سيارة عطلت في نص الطريق، السيستم بيوزع باقي الطلبات على سيارات تانية ويحسب ETAs جديدة.
👤 Dispatcher — تلقائي بعد بلاغ السائق
الـ flow: سائق يبلغ عطل ← السيستم يلاقي أقرب سائق فاضي ← يعيد توزيع الطلبات.
05
Stop-Time Estimator (ETA)
حساب وقت الوصول لكل محطة بناءً على: المسافة، متوسط وقت التفريغ، وعدد الطرود.
👤 Dispatcher يشوف — Customer بيستقبله
المعادلة: (مسافة / سرعة) + (عدد الطرود × وقت تفريغ) = ETA لكل محطة.
06
Multi-Drop Sequence Optimizer
خوارزمية بتعيد ترتيب المحطات عشان أقل مسافة إجمالية — مشابه لـ Travelling Salesman Problem.
👤 Dispatcher — تلقائي
الـ flow: قائمة المحطات ← خوارزمية nearest neighbor ← ترتيب أمثل.
07
Emergency Express Insertion
إدخال طلب عاجل في مسار شغال وإعادة حساب باقي ETAs.
👤 Dispatcher
الـ flow: طلب عاجل جديد ← Dispatcher يختار المسار ← السيستم يحقن الطلب في أنسب نقطة ← يعيد حساب الـ ETAs.
08
Driver-Vehicle Pairing Guard
السيستم بيتحقق إن رخصة السائق تسمح بقيادة السيارة المعينة.
👤 Dispatcher — تلقائي
الـ flow: Dispatcher يعين سائق لسيارة ← السيستم يقارن الرخصة بنوع السيارة ← يوافق أو يمنع.
09
Shift Transition Manager
تسليم مسار وسيارة وطلبات من سائق لآخر في منتصف الشيفت.
👤 Dispatcher
الـ flow: السائق الأول ينهي شيفته ← Dispatcher يختار سائق تاني ← كل الطلبات المتبقية تتحول للسائق الجديد.
10
Delivery Window Violation Alert
لو ETA سائق هيتأخر عن النافذة الزمنية الموعودة، تنبيه فوري للـ Dispatcher.
تلقائي ← Dispatcher يستقبل
الـ flow: حساب ETA ← مقارنة بالـ promised window ← لو هيتعدى: تنبيه + خيارات.
11
Fuel-Efficient Speed Advisor
اقتراح السرعة المثلى لكل مسار بناءً على وزن الحمولة وطول المسافة لتوفير الوقود.
👤 Dispatcher يشوف — السائق يطبق
مثال: سيارة محملة بالكامل ← اقتراح 80 كم/ساعة بدل 100 لتوفير 15% وقود.
39
Bulk Order Import (CSV/XML)
رفع قائمة الطلبات اليومية من ملف CSV أو XML جاي من نظام المبيعات.
👤 Dispatcher
الـ flow: Dispatcher يرفع الملف ← السيستم يتحقق من الأعمدة ← يستورد الطلبات ← جاهزة للتوزيع.
37
RBAC — Role-Based Access Control
تحديد مين يشوف إيه في الـ Dashboard.
👤 Fleet Manager فقط يديره
مثال: Fleet Manager: كل حاجة | Dispatcher: routing + orders بس | Mechanic: maintenance بس.
38
Historical Route Playback
مراجعة مسار كامل اتنفذ قبل كده خطوة بخطوة: مين السائق، وقت كل محطة، أي تأخيرات.
👤 Fleet Manager فقط
الاستخدام: تحقيق في شكوى ← ولا مراجعة كفاءة سائق ← ولا تحسين مسار متكرر.
40
Sustainability Report (CO2)
تقرير شهري بإجمالي انبعاثات الأسطول مع اقتراحات لتقليلها.
👤 Fleet Manager فقط
الحساب: مسافة كل سيارة × معامل انبعاثات النوع = CO2 إجمالي.
41
System Audit Trail
سجل دائم لكل تعديل يدوي في السيستم: مين غيّر إيه، امتى، وإيه كانت القيمة قبل وبعد.
👤 Fleet Manager فقط يشوفه
مثال: Dispatcher غيّر حالة طلب يدوياً ← السجل يحفظ: user_id + timestamp + القيمة القديمة + الجديدة.
42
Database Integrity Manager
أرشفة تلقائية للمسارات القديمة وسجلات الصيانة المكتملة للحفاظ على سرعة السيستم.
👤 Fleet Manager — تشغيل يدوي أو جدولة تلقائية
الـ flow: كل فترة ← نقل البيانات الأقدم من X شهر لجداول archive ← الـ main tables تبقى خفيفة.

N
Mechanic Login & Auth
جديد
الميكانيكي بيعمل login عشان يشوف البلاغات المعينة له والسيارات المطلوب صيانتها.
👤 Mechanic + Fleet Manager
ليه جديد؟ مش مذكور في الـ 42 خالص، بس من غيره الميكانيكي مش قادر يدخل على التطبيق.
23
Odometer-Driven Service Alerts
السيستم بيتابع عداد كل سيارة وبيبعت تنبيه تلقائي لما تحتاج زيت أو جاطوك أو أي صيانة دورية.
تلقائي ← Fleet Manager + Mechanic
مثال: كل 5000 كم ← تنبيه "تغيير زيت" ← Mechanic يستقبل البلاغ ← يحجز موعد.
24
Fuel Expense Audit
مقارنة فواتير الوقود الفعلية بالمسافة اللي اتسافرت — لو في فرق كبير ممكن يكون في تسريب أو سرقة.
👤 Fleet Manager
الـ flow: السائق يرفع فاتورة وقود ← السيستم يقارنها بالـ GPS ← لو أكتر من X% فرق: تنبيه.
25
Vehicle Out-of-Service Lockout
لما سيارة بتدخل الصيانة، بتتقفل تلقائياً من أي تعيين جديد في الـ Dashboard.
👤 Mechanic يحددها — Dispatcher مش يقدر يعينها
الـ flow: Mechanic يغير حالة السيارة لـ "In Repair" ← السيستم يحذفها من قائمة السيارات المتاحة.
26
Part Replacement Lifecycle
تتبع عمر كل قطعة في السيارة: فرامل، بطارية، فلتر هواء — ومتى اتغيرت آخر مرة.
👤 Mechanic يحدّث — Fleet Manager يراجع
الـ flow: بعد كل إصلاح ← Mechanic يسجل القطع المتغيرة ← السيستم يحسب العمر المتبقي.
27
Annual Inspection & Insurance
تنبيهات قبل انتهاء التأمين أو الفحص الدوري السنوي لكل سيارة.
تلقائي ← Fleet Manager
مثال: 30 يوم قبل انتهاء التأمين ← تنبيه للـ Fleet Manager ← يجدد.
28
Fuel Efficiency Comparator
مقارنة كفاءة استهلاك الوقود بين السيارات لمعرفة أيها أكثر اقتصاداً.
👤 Fleet Manager
الناتج: جدول أو رسم بياني: سيارة A = 12 كم/لتر، سيارة B = 9 كم/لتر ← قرار التحديث.
29
Maintenance Cost-to-Value
لو تكلفة إصلاح سيارة بتتجاوز نسبة معينة من قيمتها السوقية، السيستم بيقترح سحبها من الخدمة.
👤 Fleet Manager
المعادلة: تكلفة الإصلاح ÷ القيمة السوقية. لو أكتر من 40% ← "يُنصح بالاستبدال".
30
Technician Assignment Hub
تعيين الميكانيكي المناسب على بلاغ صيانة معين، داخلي أو خارجي.
👤 Fleet Manager يعين ← Mechanic يستقبل
الـ flow: بلاغ صيانة ← Fleet Manager يختار ميكانيكي ← الميكانيكي يشوف البلاغ في تطبيقه.
31
Spare Parts Inventory
تتبع مخزون قطع الغيار في الجراج: زيوت، فلاتر، لمبات، فرامل.
👤 Mechanic يستخدم — Fleet Manager يراجع
الـ flow: Mechanic يطلب قطعة ← السيستم يخصمها من المخزن ← لو وصل للحد الأدنى: تنبيه إعادة طلب.
32
Emergency Roadside Assistance
إرسال مساعدة طارئة لموقع السائق العاطل بالضبط.
👤 Fleet Manager / Dispatcher يقرر ← Mechanic ينفّذ
الـ flow: السائق يبلغ عطل ← موقع GPS تلقائي ← Fleet Manager يبعت أقرب ميكانيكي.
